DML Commands (Data Manipulation Language)

DML commands are used to manipulate the actual data(records) in the database. The most frequently used DML statements are SELECT, INSERT, UPDATE, and DELETE.

1️⃣ INSERT Statement

Purpose: The INSERT statement is used to add new records (rows) to a table.

Syntax:

INSERT INTO table_name (column1, column2, column3, ...) VALUES (value1, value2, value3, ...);
📌 Example:
INSERT INTO EMPLOYEES (EMP_ID, EMP_NAME, SALARY, DEPT_ID) VALUES (107, 'John Doe', 50000, 'IT');

2️⃣ UPDATE Statement

Purpose: The UPDATE statement is used to modify existing records in a table.

Syntax:

UPDATE table_name SET column1 = value1, column2 = value2, ... WHERE condition;
📌 Example:
UPDATE EMPLOYEES SET SALARY = 55000, DEPT_ID = 'HR' WHERE EMP_ID = 107;
⚠️ Important: Always use a WHERE clause in UPDATE statements to specify which records to update. Without WHERE, ALL records will be updated!

3️⃣ DELETE Statement

Purpose: The DELETE statement is used to remove records from a table.

Syntax:

DELETE FROM table_name WHERE condition;
📌 Example:
DELETE FROM EMPLOYEES WHERE EMP_ID = 107;

Key Differences:

Statement Purpose Effect on Columns
INSERT Adds new rows Adds all column values
UPDATE Modifies existing rows Changes specific column values
DELETE Removes entire rows Removes entire row structure

SQL Operators

SQL operators are used to specify conditions in WHERE clauses and to combine multiple conditions with multiple queries. They are essential for filtering and comparing data.

1️⃣ Arithmetic Operators

Operator Description Example
+ Addition SELECT SALARY + 5000
- Subtraction SELECT SALARY - 2000
* Multiplication SELECT PRICE * 1.10
/ Division SELECT SALARY / 12
% Modulus SELECT QUANTITY % 5

2️⃣ Comparison Operators

Operator Description Example
= Equal to WHERE DEPT_ID = 'IT'
!=, <> Not equal to WHERE STATUS != 'INACTIVE'
> Greater than WHERE SALARY > 50000
< Less than WHERE AGE < 30
>= Greater than or equal WHERE PRICE >= 1000
<= Less than or equal WHERE QUANTITY <= 100

3️⃣ Logical Operators

Operator Purpose Example
AND All conditions must be TRUE WHERE DEPT = 'IT' AND SALARY > 50000
OR At least one condition must be TRUE WHERE STATUS = 'ACTIVE' OR STATUS = 'PENDING'
NOT Negates a condition WHERE NOT STATUS = 'INACTIVE'

4️⃣ Special Operators

BETWEEN: Selects values within a given range (inclusive)
WHERE AGE BETWEEN 25 AND 35
IN: Allows multiple values in a WHERE clause
WHERE DEPT_ID IN ('IT', 'HR', 'FINANCE')
LIKE: Searches for a specified pattern
WHERE EMP_NAME LIKE 'J%' (starts with J)
IS NULL: Tests for NULL values
WHERE MANAGER_ID IS NULL

SQL Functions

SQL functions are built-in functions that operate on data and return results. Functions can be aggregate functions (work on multiple rows) or scalar functions (work on individual values).

📊 Aggregate Functions

These functions return a single result row based on groups of rows:

Function Description Example
COUNT() Returns the number of rows SELECT COUNT(*) FROM EMPLOYEES
SUM() Returns the sum of numeric values SELECT SUM(SALARY) FROM EMPLOYEES
AVG() Returns the average value SELECT AVG(SALARY) FROM EMPLOYEES
MAX() Returns the maximum value SELECT MAX(SALARY) FROM EMPLOYEES
MIN() Returns the minimum value SELECT MIN(SALARY) FROM EMPLOYEES

🔤 String Functions

Function Description Example
UPPER() Converts to uppercase SELECT UPPER(EMP_NAME) FROM EMPLOYEES
LOWER() Converts to lowercase SELECT LOWER(EMP_NAME) FROM EMPLOYEES
LENGTH() Returns length of string SELECT LENGTH(EMP_NAME) FROM EMPLOYEES
SUBSTR() Extracts substring SELECT SUBSTR(EMP_NAME, 1, 3) FROM EMPLOYEES
TRIM() Removes leading/trailing spaces SELECT TRIM(EMP_NAME) FROM EMPLOYEES
REPLACE() Replaces substring SELECT REPLACE(EMP_NAME, 'John', 'Jane')

🔢 Numeric Functions

Function Description Example
ABS() Returns absolute value SELECT ABS(-50) returns 50
ROUND() Rounds to specified decimals SELECT ROUND(SALARY, 2)
CEIL() Rounds up to integer SELECT CEIL(45.3) returns 46
FLOOR() Rounds down to integer SELECT FLOOR(45.9) returns 45
MOD() Returns remainder SELECT MOD(10, 3) returns 1

📅 Date Functions

Function Description Example
SYSDATE Returns current date SELECT SYSDATE FROM DUAL
TRUNC() Truncates date to specified unit SELECT TRUNC(SYSDATE, 'MM')
MONTHS_BETWEEN() Returns months between two dates SELECT MONTHS_BETWEEN(SYSDATE, DOB)

SQL Joins

A JOIN clause is used to combine rows from two or more tables, based on a related column between them. Joins allow you to retrieve data from multiple tables in a single query.

1️⃣ INNER JOIN

Purpose: Returns records that have matching values in both tables.

Syntax:

SELECT columns FROM table1 INNER JOIN table2 ON table1.common_column = table2.common_column;
📌 Example:
SELECT E.EMP_NAME, D.DEPT_NAME
FROM EMPLOYEES E
INNER JOIN DEPARTMENTS D
ON E.DEPT_ID = D.DEPT_ID;

2️⃣ LEFT JOIN (LEFT OUTER JOIN)

Purpose: Returns all records from the left table, and the matched records from the right table. Non-matching rows show NULL for right table columns.

Syntax:

SELECT columns FROM table1 LEFT JOIN table2 ON table1.common_column = table2.common_column;
📌 Example:
SELECT E.EMP_NAME, D.DEPT_NAME
FROM EMPLOYEES E
LEFT JOIN DEPARTMENTS D
ON E.DEPT_ID = D.DEPT_ID;

3️⃣ RIGHT JOIN (RIGHT OUTER JOIN)

Purpose: Returns all records from the right table, and the matched records from the left table. Non-matching rows show NULL for left table columns.

Syntax:

SELECT columns FROM table1 RIGHT JOIN table2 ON table1.common_column = table2.common_column;

4️⃣ FULL OUTER JOIN

Purpose: Returns records when there is a match in either left or right table. Combines the results of both LEFT and RIGHT JOINs.

Syntax:

SELECT columns FROM table1 FULL OUTER JOIN table2 ON table1.common_column = table2.common_column;

5️⃣ CROSS JOIN

Purpose: Returns the Cartesian product (all possible combinations) of two tables.

Syntax:

SELECT columns FROM table1 CROSS JOIN table2;

Join Types Comparison:

Join Type Returns Best Used For
INNER JOIN Matching records only When you need data present in both tables
LEFT JOIN All left + matched right records When you want all records from left table
RIGHT JOIN All right + matched left records When you want all records from right table
FULL OUTER JOIN All records from both tables When you need all data from both tables
CROSS JOIN All combinations Rarely used; creates massive result sets

Subqueries & Views

A subquery is a query within another query. The query that contains the subquery is called the outer query or main query. Views are virtual tables based on a SELECT query.

🔍 Subqueries (Inner Queries)

Purpose: A subquery is used to return data that will be used by the main query as a condition to further restrict the data to be retrieved.

Types of Subqueries:

1. Single Row Subquery

SELECT columns FROM table1 WHERE column = ( SELECT column FROM table2 WHERE condition );
📌 Example:
SELECT EMP_NAME, SALARY
FROM EMPLOYEES
WHERE SALARY > (SELECT AVG(SALARY) FROM EMPLOYEES);

2. Multiple Row Subquery

SELECT columns FROM table1 WHERE column IN ( SELECT column FROM table2 WHERE condition );
📌 Example:
SELECT EMP_NAME
FROM EMPLOYEES
WHERE DEPT_ID IN (SELECT DEPT_ID FROM DEPARTMENTS WHERE LOCATION = 'New York');

3. Correlated Subquery

SELECT columns FROM table1 outer_alias WHERE condition IN ( SELECT column FROM table2 WHERE table2.column = outer_alias.column );

👁️ Views

Purpose: A view is a virtual table created based on the result-set of a SELECT statement. Views contain rows and columns just like a real table, but the data is derived from one or more tables.

Syntax to Create a View:

CREATE VIEW view_name AS SELECT columns FROM table_name WHERE condition;
📌 Example:
CREATE VIEW HIGH_SALARY_EMPLOYEES AS
SELECT EMP_ID, EMP_NAME, SALARY
FROM EMPLOYEES
WHERE SALARY > 60000;

Querying a View:

SELECT columns FROM view_name WHERE condition;

Alter a View:

ALTER VIEW view_name AS SELECT new_columns FROM table_name WHERE new_condition;

Drop a View:

DROP VIEW view_name;

Advantages of Views:

  • Simplifies complex queries
  • Provides security by restricting access to specific columns/rows
  • Hides complexity from users
  • Acts as a stored SELECT statement
  • Maintains consistency of data

Sequences & Synonyms

Sequences are used to generate unique numbers automatically. Synonyms are alternative names for objects in the database.

🔢 Sequences

Purpose: A sequence is a database object that generates a series of unique numeric values. It's commonly used to generate primary key values.

Create a Sequence:

CREATE SEQUENCE sequence_name [START WITH start_value] [INCREMENT BY increment_value] [MAXVALUE max_value] [MINVALUE min_value] [CYCLE|NOCYCLE] [CACHE cache_size];
📌 Example:
CREATE SEQUENCE EMP_SEQ
START WITH 1001
INCREMENT BY 1
MAXVALUE 9999
NOCYCLE
CACHE 20;

Using Sequences:

NEXTVAL: Returns the next value from the sequence
CURRVAL: Returns the current value of the sequence
📌 Example:
INSERT INTO EMPLOYEES (EMP_ID, EMP_NAME, SALARY)
VALUES (EMP_SEQ.NEXTVAL, 'John Doe', 50000);

Alter a Sequence:

ALTER SEQUENCE sequence_name [INCREMENT BY increment_value] [MAXVALUE max_value] [MINVALUE min_value];

Drop a Sequence:

DROP SEQUENCE sequence_name;

🔗 Synonyms

Purpose: A synonym is an alternative name for a schema object (table, view, sequence, etc.). It provides a shorter or more convenient name for objects.

Create a Synonym:

CREATE SYNONYM synonym_name FOR object_owner.object_name;
📌 Example:
CREATE SYNONYM EMP FOR HR.EMPLOYEES;
-- Now you can use: SELECT * FROM EMP;

Public vs Private Synonyms:

-- Public Synonym (accessible by all users) CREATE PUBLIC SYNONYM EMP FOR HR.EMPLOYEES; -- Private Synonym (accessible by creator only) CREATE SYNONYM EMP FOR HR.EMPLOYEES;

Drop a Synonym:

DROP SYNONYM synonym_name; DROP PUBLIC SYNONYM synonym_name;

PL/SQL Basics

PL/SQL (Procedural Language/Structured Query Language) is Oracle's procedural extension of standard SQL. It enables writing of complete applications and allows combining SQL with procedural statements.

📝 PL/SQL Block Structure

DECLARE -- Variable declarations variable_name datatype; BEGIN -- Executable statements SQL statements; PL/SQL statements; EXCEPTION -- Error handling WHEN exception_type THEN -- Handle exception END; /

🔢 Data Types

Category Data Type Description
Character CHAR, VARCHAR2 Fixed or variable-length strings
Numeric NUMBER, INTEGER, DECIMAL Numeric values
Date DATE, TIMESTAMP Date and time values
Boolean BOOLEAN TRUE, FALSE, or NULL

💾 Variable Declaration

DECLARE emp_name VARCHAR2(50); emp_salary NUMBER(10,2); hire_date DATE; is_active BOOLEAN; BEGIN emp_name := 'John Doe'; emp_salary := 50000.00; hire_date := SYSDATE; is_active := TRUE; END; /

🔄 Control Structures

IF-THEN-ELSE Statement:

IF condition THEN statements; ELSIF condition THEN statements; ELSE statements; END IF;
📌 Example:
IF salary > 50000 THEN
    bonus := salary * 0.10;
ELSIF salary > 30000 THEN
    bonus := salary * 0.05;
ELSE
    bonus := 0;
END IF;

LOOP Statements:

-- Simple LOOP LOOP statements; EXIT WHEN condition; END LOOP; -- FOR LOOP FOR counter IN start_value .. end_value LOOP statements; END LOOP; -- WHILE LOOP WHILE condition LOOP statements; END LOOP;

⚠️ Exception Handling

BEGIN -- Executable statements INSERT INTO EMPLOYEES VALUES (...); EXCEPTION WHEN DUP_VAL_ON_INDEX THEN DBMS_OUTPUT.PUT_LINE('Duplicate value error'); WHEN TOO_MANY_ROWS THEN DBMS_OUTPUT.PUT_LINE('Too many rows found'); WHEN OTHERS THEN DBMS_OUTPUT.PUT_LINE('Unknown error occurred'); END; /

Stored Procedures & Functions

Stored procedures and functions are reusable PL/SQL programs that perform specific tasks. They are stored in the database and can be called by other programs.

🛠️ Stored Procedures

Purpose: A stored procedure is a named PL/SQL block that performs a specific task. It can accept parameters and perform complex operations.

Create a Procedure:

CREATE PROCEDURE procedure_name ( parameter1 [IN|OUT|IN OUT] datatype, parameter2 [IN|OUT|IN OUT] datatype ) IS BEGIN -- Procedure body -- SQL and PL/SQL statements END procedure_name; /
📌 Example:
CREATE PROCEDURE add_employee (
    p_emp_id IN NUMBER,
    p_emp_name IN VARCHAR2,
    p_salary IN NUMBER
) IS
BEGIN
    INSERT INTO EMPLOYEES (EMP_ID, EMP_NAME, SALARY)
    VALUES (p_emp_id, p_emp_name, p_salary);
    COMMIT;
END add_employee;
/

Execute a Procedure:

-- Using EXECUTE statement EXECUTE procedure_name(param1, param2); -- Using CALL statement CALL procedure_name(param1, param2); -- Inside PL/SQL block BEGIN procedure_name(param1, param2); END; /

📊 Stored Functions

Purpose: A function is similar to a procedure, but it MUST return a single value. Functions are typically used in SQL statements.

Create a Function:

CREATE FUNCTION function_name ( parameter1 datatype, parameter2 datatype ) RETURN return_datatype IS -- Variable declarations BEGIN -- Function body -- Must include a RETURN statement RETURN value; END function_name; /
📌 Example:
CREATE FUNCTION get_annual_salary (p_monthly_salary NUMBER) RETURN NUMBER IS
BEGIN
    RETURN (p_monthly_salary * 12);
END get_annual_salary;
/

Call a Function:

-- In SQL statement SELECT EMP_NAME, get_annual_salary(SALARY) AS ANNUAL_SALARY FROM EMPLOYEES; -- In PL/SQL block DECLARE annual_sal NUMBER; BEGIN annual_sal := get_annual_salary(50000); DBMS_OUTPUT.PUT_LINE('Annual Salary: ' || annual_sal); END; /

Procedure vs Function:

Aspect Procedure Function
Return Value Optional (0, 1, or multiple) Must return exactly one value
Parameters IN, OUT, IN OUT Only IN parameters
Used in SQL Cannot be used in SQL Can be used in SQL statements
Calling CALL or EXECUTE Called like a function in SQL

Triggers & Cursors

Triggers are special stored procedures that execute automatically in response to specific events on a database table. Cursors are used to process rows returned by a query.

🔔 Triggers

Purpose: A trigger is a special type of stored procedure that automatically executes (or "fires") in response to specific events on a particular table in the database.

Trigger Events:

  • INSERT - Fires when a new row is inserted
  • UPDATE - Fires when an existing row is updated
  • DELETE - Fires when a row is deleted

Trigger Timing:

  • BEFORE - Fires before the triggering event
  • AFTER - Fires after the triggering event

Create a Trigger Syntax:

CREATE [OR REPLACE] TRIGGER trigger_name {BEFORE | AFTER} {INSERT | UPDATE | DELETE} ON table_name [FOR EACH ROW] BEGIN -- Trigger body -- PL/SQL statements END trigger_name; /
📌 Example: AFTER INSERT Trigger
CREATE OR REPLACE TRIGGER emp_audit_insert
AFTER INSERT ON EMPLOYEES
FOR EACH ROW
BEGIN
    INSERT INTO EMP_AUDIT (EMP_ID, ACTION, ACTION_DATE)
    VALUES (:NEW.EMP_ID, 'INSERTED', SYSDATE);
END emp_audit_insert;
/
💡 Important: Use :NEW to reference new values after INSERT/UPDATE, and :OLD to reference old values before UPDATE/DELETE.

📋 Cursors

Purpose: A cursor is a memory area that stores the context of the query. Cursors are used to process rows returned by a query one at a time.

Types of Cursors:

1. Implicit Cursors

Automatically created by Oracle for DML statements (INSERT, UPDATE, DELETE, SELECT). No need to declare or open explicitly.

2. Explicit Cursors

-- Declare cursor DECLARE CURSOR cursor_name IS SELECT statement; BEGIN -- Open cursor OPEN cursor_name; -- Fetch rows FETCH cursor_name INTO variables; -- Process rows -- ... -- Close cursor CLOSE cursor_name; END; /
📌 Example: Explicit Cursor
DECLARE
    CURSOR emp_cursor IS
        SELECT EMP_ID, EMP_NAME, SALARY FROM EMPLOYEES;
    v_emp_id NUMBER;
    v_emp_name VARCHAR2(50);
    v_salary NUMBER;
BEGIN
    OPEN emp_cursor;
    LOOP
        FETCH emp_cursor INTO v_emp_id, v_emp_name, v_salary;
        EXIT WHEN emp_cursor%NOTFOUND;
        DBMS_OUTPUT.PUT_LINE(v_emp_id || ' ' || v_emp_name);
    END LOOP;
    CLOSE emp_cursor;
END;
/

Cursor Attributes:

Attribute Description
%FOUND TRUE if FETCH returned a row
%NOTFOUND TRUE if FETCH did not return a row
%ROWCOUNT Number of rows fetched so far
%ISOPEN TRUE if cursor is open
